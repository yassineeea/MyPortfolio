Data Deduplication & Profiling

The purpose of this project is to ensure a smooth data migration from SAP ECC to SAP HANA while maintaining high data quality. By developing comprehensive data deduplication and profiling solutions using SAP BODS and SAP Information Steward, we aim to enhance data accuracy and consistency.

Additional Project Information

Here you can find detailed information about the project, including Data extraction, Data deduplication, Data profiling and more.

Tools Used


  • SAP Business Object Data Services (BODS): ETL tool for data integration, data quality, data profiling, and data cleansing.
  • SAP Information Steward (IS): Tool for data profiling, data quality monitoring, metadata management, and data governance.
  • Power BI: Business analytics tool for data visualization, reporting, and interactive dashboards creation.
  • Hana Database: In-memory, column-oriented, relational database management system for real-time data processing and analytics.

DATA DESCRIPTION


  • The dataset I analyzed comprised 13 columns and 21,748 rows, focusing on customer data from the SAP system, stored in the HANA database. It included customer information and demographics related to various business operations.

REFERENCE :

    The dataset was extracted from Hana DB of SANOFI Client.


Data Extraction


  • In the data extraction phase, we connected SAP ECC to the HANA database and created a job in SAP BODS to extract essential customer data (KNA1).
    We focused on customers with open items, recent sales documents, or cleared items, following the client's specific requirements to ensure only relevant data was migrated.

Data Profiling


    During the data profiling phase, we implemented various types of profiling, including uniqueness, Redondancy and columns profiling.
  • 1. Uniquness Profiling: Using Information Steward, we conducted uniqueness profiling and found that a majority of data in the 'NAME' and 'ADDRESS' columns are unique, indicating good data quality. However, there is still a significant proportion of non-unique data that may require attention. Specifically, we identified that 34.38% of entries in the 'NAME' and 'ADDRESS' columns were exact duplicates
  • 2. Redondancy Profiling: As part of our comprehensive database analysis, we conducted redundancy profiling. This step allowed us to assess the degree of overlap and duplication of data values between two sets of columns.
  • 3. Columns Profiling: During the column profiling phase, we accurately determined the values and characteristics of data elements. This included identifying the string lengths for each column, enabling us to verify the consistency and compliance of the entered data.

DATA DEDUPLICATION


  • When data is entered manually, it is very likely that the same record will be entered twice with slight differences in certain attributes. Our mission is to implement a duplicate identification solution based on the following two criteria:
     NAME1: customer's name.
    ADDRESS: customer's address.
    Therefore, we used two approaches for this data deduplication solution.

    As a first approach, we used the MatchBatch component of BODS. For this, after selecting the matching criteria of Name and Address on which the program will base its comparison, we configured the matching options according to the chosen data deduplication method.
    We have 3 different methods based on the options:
  • Rule Based.
  • Weighted Score.
  • Combination.
    • Among the matching options, we have Match score, No match score and contribution.

  • After exploring the deduplication results with SAP BODS, we will now turn to another approach using the Data Cleansing Advisor component of SAP Information Steward. Once the program finds the matching clusters, it provides an interface that displays the matching records with different confidence levels (High, Medium, and Low).

DATA ANALYSIS AND VISUALIZATION


  • We have made the deduplication results extracted from BODS in the previous section much more readable for the client using dashboards developed with Microsoft's POWER BI tool. The main dashboard indicators include:

    1. Number of Duplicates: Displays the total number of duplicates present in the system, indicating the level of data repetition.
    2. Clusters: Lists or visually represents the clusters identified in the data. Each cluster groups similar records based on defined criteria.
    3. Location by Break Key: Shows the geographic distribution or location of records within each cluster, based on a specific break key.

FINDINGS :


  • 1. Comparison between BODS & IS:
         The choice between SAP BODS and SAP Information Steward for data deduplication depends on the specific needs of your organization.
    If you require maximum flexibility and precision, SAP BODS may be the best choice, but it requires a certain level of technical expertise and in-depth knowledge of the tool.
    However, if you prioritize the visibility of results, SAP Information Steward might be more suitable.
  • 2. Data Quality Improvement:
    By implementing robust data profiling and deduplication measures, the overall quality of the dataset improved significantly. This led to better decision-making and more reliable insights.
  • 3. Enhanced Data Governance:
    The use of SAP Information Steward helped in establishing better data governance practices, ensuring data accuracy, consistency, and compliance across the organization.
